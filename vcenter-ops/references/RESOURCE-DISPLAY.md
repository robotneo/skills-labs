# 资源展示格式规范

> 从 SKILL.md 展开的独立指令集展示格式。

## 触发条件

用户要求查看/展示 vCenter 资源信息。

## 执行流程

1. `cache_manager.py --status` 检查缓存
2. 无缓存 → `--refresh` → 有缓存 → 按需读取
3. 用户说"刷新" → 强制重查

## 按需展示

| 用户说 | 展示内容 | 提取命令 |
|--------|---------|---------|
| "展示全部"/"巡检" | 全部 | `--summary` + 各 section |
| "模板" | 仅模板 | `--section templates` |
| "存储" | 仅存储 | `--section datastores --top 10` |
| "网络" | 仅网络 | `--section networks` |
| "集群" | 仅集群 | `--summary` 中集群部分 |
| "宿主机" | 评分 Top5 | `--section hosts --cluster xxx` |

## 全部展示格式

```
📁 数据中心
  - Datacenter

🏗️ 计算集群
  - MD1200 | SC4020 | vSAN-A

📦 克隆模板
| # | 名称 | 系统 | 规格 |
|---|------|------|------|
| 1 | Ubuntu20.04-2c4g60g-lvm2 | Ubuntu 64-bit | 2C/4G/60G |
...

💾 存储池（主要）
| # | 名称 | 总容量 | 可用 | 可用率 |
|---|------|-------|------|-------|
| 1 | vsanDatastore | 29.8TB | 14.4TB | 48.3% ✅ |
...
> 共 N 个存储池，其余多为各宿主机本地数据盘。

🌐 网络
| # | 名称 |
|---|------|
| 1 | yewu |
...
```

## 单类展示格式

用户只问某一类时，只输出该类别，不加其他区块。同样使用编号表格。

## 展示后引导

资源展示完毕后，**必须**追加：

```
---
需要创建虚拟机？请提供：
  名称: TEST-VM-01 | 集群/模板/存储/网络: 编号或名称
  硬件: 4C/8G/100G（默认，可改）| IP: 从可用列表选择

示例: "名称 TEST-VM-01，集群 vSAN-A，模板 3，存储 vsanDatastore，网络 yewu"
```

## 关键规则

- 存储 `is_low_space=true` 标注 ⚠️
- 宿主机按评分排序 Top5
- 本地数据盘（500GB级、IP开头如43.x-data）合并为摘要
- 模板/存储/网络必须编号表格
- 展示完毕后必须引导创建虚拟机
