# 三级权限控制体系

> 从 SKILL.md 展开的权限模型与确认模板。

## 权限模型

| 级别 | 标识 | 操作 | 确认次数 | 审批 |
|------|------|------|---------|------|
| confirm | 🔵 普通 | clone_vm, get_vm, list_all, snapshot | 1 次 | 无 |
| double_confirm | 🟡 敏感 | power_vm | 2 次 | 无 |
| approval | 🔴 高风险 | delete_vm | 2 次 | 预留接口 |

## 权限配置来源

`config.yaml` → `permission.action_map`

## 🟡 Level: double_confirm（电源管理）

### 第一次确认

```
📋 电源操作确认
━━━━━━━━━━━━━━━━━━━━
目标: PROD-DB-01 | 当前: poweredOn | 操作: 关机
━━━━━━━━━━━━━━━━━━━━
权限级别: 🟡 敏感操作

第一次确认: 确认关机？（回复"确认"）
```

### 第二次确认（含风险提醒）

```
⚠️ 风险提醒
━━━━━━━━━━━━━━━━━━━━
操作: 关闭 PROD-DB-01
风险: 所有业务中断，请确认已通知相关方
影响: 连接该 VM 的用户断开连接
回滚: power_vm --state on 重新开机
━━━━━━━━━━━━━━━━━━━━

第二次确认: 确定执行？（回复"确认执行"）
```

## 🔴 Level: approval（删除虚拟机）

### 第一次确认

```
📋 删除操作确认
━━━━━━━━━━━━━━━━━━━━
目标: PROD-OLD-01
位置: DC-01 / Cluster-A / esxi-01
规格: 4C / 8GB / 100GB
━━━━━━━━━━━━━━━━━━━━
权限级别: 🔴 高风险操作

第一次确认: 确认永久删除？（回复"确认"）
```

### 第二次确认（含风险提醒 + 审批预留）

```
⚠️ 风险提醒
━━━━━━━━━━━━━━━━━━━━
操作: 永久删除 PROD-OLD-01（不可恢复）
风险: 所有数据永久丢失（磁盘/快照/配置）
影响: 从 vCenter 清单完全移除
回滚: ❌ 不可回滚（除非有外部备份）
━━━━━━━━━━━━━━━━━━━━

📌 审批接口待接入（当前自动跳过）

第二次确认: 确定删除？（回复"确认删除"）
```

### 删除执行细节

1. `handler.py --action delete_vm --hostname "VM名称"`
2. `remove_vm()` 自动处理电源：
   - 开机 → 先 PowerOff → 再 Destroy
   - 关机 → 直接 Destroy
3. 独立 `power_vm` 操作不受影响
