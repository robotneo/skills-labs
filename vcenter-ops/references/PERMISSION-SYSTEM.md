# 三级权限控制体系

> 从 SKILL.md 展开的权限模型与确认模板。

## 权限模型

| 级别 | 标识 | 操作 | 确认次数 | 审批 |
|------|------|------|---------|------|
| confirm | 🔵 普通 | clone_vm, get_vm, list_all, snapshot | 1 次 | 无 |
| double_confirm | 🟡 敏感 | power_vm | 2 次 | 无 |
| approval | 🔴 高风险 | delete_vm | 2 次 | 预留接口 |

## 权限配置来源

`config.yaml` → `permission.action_map`

## 确认机制

所有确认环节使用 **单字母快捷回复**，降低用户操作成本：

| 用户输入 | 等效操作 |
|----------|---------|
| `Y` / `y` / `是` / `确认` | 确认执行 |
| `N` / `n` / `否` / `取消` | 取消操作 |

> Agent 输出确认卡片时，末尾固定显示快捷提示：`回复 Y 确认 / N 取消`

## 🔴 删除操作安全策略（强制执行）

> ⚠️ 以下规则对所有删除类动作 **无条件强制生效**，Agent 不得绕过、不得跳过、不得自行解释。

### 适用范围

以下关键词均视为删除类动作，全部适用本策略：
- 中文：删除、销毁、毁灭、移除、清除、抹除
- 英文：delete、destroy、remove、erase、purge
- 缩写/别名：rm、del

适用对象不限于虚拟机，包括但不限于：虚拟机、快照、磁盘、网络、资源池等一切 vCenter 资源。

### 规则 1：二次确认（强制）

所有删除操作 **必须经过两次确认**，不允许一次确认后直接执行。

- 第一次确认：展示操作详情，用户回复 Y 后进入第二次
- 第二次确认：展示风险提醒，用户再次回复 Y 后才可执行
- 任意一次回复 N → 立即取消，不执行

### 规则 2：禁止全量删除（强制）

**以下请求一律拒绝，不执行、不确认、直接驳回：**

| 禁止请求 | 示例 |
|----------|------|
| 未指定目标 | "帮我删除虚拟机"、"删除所有VM" |
| 全量/批量泛指 | "删除全部"、"清空所有虚拟机"、"把集群里的都删了" |
| 通配符/模糊匹配 | "删除所有测试机"、"删除 test-开头的VM" |
| 按条件批量删除 | "删除关机的虚拟机"、"删除没用的VM" |
| 删除整集群/整主机 | "删除 Cluster-A 所有VM"、"清空 esxi-01" |

**Agent 行为**：检测到上述模式时，立即回复：

```
🚫 操作被拒绝：不允许全量/批量删除。

请指定具体删除目标（虚拟机名称或 IP），例如：
- 删除虚拟机 PROD-OLD-01
- 删除 172.17.40.5
```

### 规则 3：必须指定明确目标（强制）

执行删除前，用户 **必须明确指定** 以下之一：

| 可接受的目标格式 | 示例 |
|----------------|------|
| 虚拟机名称（精确） | `PROD-OLD-01`、`172.17.40.5-PROD-WEB-05` |
| 虚拟机 IP（精确） | `172.17.40.5` |
| 多个明确目标（逐个指定） | `PROD-OLD-01、PROD-OLD-02` |

**不可接受的目标格式**（等同规则 2，一律拒绝）：

| 不可接受 | 原因 |
|---------|------|
| 通配符 `*` / `?` | 范围不可控 |
| 正则表达式 | 范围不可控 |
| 前缀匹配 "test-*" | 可能命中非预期目标 |
| 按状态筛选 "关机的" | 条件不可控 |
| 空目标 | 未指定 |

**Agent 行为**：用户未指定明确目标时，主动询问并引导用户逐个指定：

```
请指定要删除的虚拟机（名称或 IP）：
一次可指定多个，用逗号分隔，例如：
PROD-OLD-01, 172.17.40.10
```

### Agent 校验清单（每次删除前必须执行）

```
□ 1. 动作是否属于删除类？（关键词匹配）
□ 2. 用户是否指定了明确的删除目标？（名称或 IP）
□ 3. 是否包含全量/批量/通配符模式？（有则拒绝）
□ 4. 是否完成了第一次确认？（Y）
□ 5. 是否完成了第二次确认（含风险提醒）？（Y）
□ 6. 全部通过 → 执行删除
   任意一条未通过 → 拒绝或等待
```


## 🔴 Level: approval（删除虚拟机）

> ⚠️ 删除操作同时受上方「删除操作安全策略」约束，以下为确认模板。

### 第一次确认

```
📋 删除操作确认
━━━━━━━━━━━━━━━━━━━━
目标: PROD-OLD-01
位置: DC-01 / Cluster-A / esxi-01
规格: 4C / 8GB / 100GB
━━━━━━━━━━━━━━━━━━━━
权限级别: 🔴 高风险操作

第一次确认：回复 Y 确认 / N 取消
```

### 第二次确认（含风险提醒 + 审批预留）

```
⚠️ 风险提醒
━━━━━━━━━━━━━━━━━━━━
操作: 永久删除 PROD-OLD-01（不可恢复）
风险: 所有数据永久丢失（磁盘/快照/配置）
影响: 从 vCenter 清单完全移除
回滚: ❌ 不可回滚（除非有外部备份）
━━━━━━━━━━━━━━━━━━━━

📌 审批接口待接入（当前自动跳过）

第二次确认：回复 Y 删除 / N 取消
```

### 删除执行细节

1. `handler.py --action delete_vm --hostname "VM名称"`
2. `remove_vm()` 自动处理电源：
   - 开机 → 先 PowerOff → 再 Destroy
   - 关机 → 直接 Destroy
3. 独立 `power_vm` 操作不受影响
